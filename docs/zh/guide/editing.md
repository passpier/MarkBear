# 編輯

無論是匯入的文件還是新建的文件，一旦開啟後，Pourdown 提供兩種編輯方式。

## 視覺化（WYSIWYG）編輯

預設檢視模式，由 [Tiptap](https://tiptap.dev/) 編輯器驅動。所見即所得地撰寫 Markdown：
輸入 `# ` 產生標題、`- ` 產生項目符號、用 `**` 包住文字產生粗體 —— 即時呈現，
不需要處理原始符號。支援豐富格式：粗體、斜體、刪除線、巢狀清單、表格、
帶語法高亮的程式碼區塊，以及引言區塊。

## 原始碼模式

隨時可從工具列切換至原始 Markdown 文字。當你想查看或手動編輯確切的 Markdown、
從別處貼上 Markdown 內容，或想確認實際會儲存到硬碟的內容時很有用。

## 檔案管理

透過原生系統對話框開啟、儲存與管理你的 Markdown 檔案。側欄
（可分別切換：檔案、大綱或隱藏）讓你能瀏覽並切換不同的開啟文件。

## 尋找與取代

文件內搜尋並支援取代（透過常見的尋找／取代快捷鍵存取），
當你需要在整個文件資料夾中尋找內容時，也可以在側欄進行跨檔案搜尋。

## 自動儲存

除了明確按下 `Cmd/Ctrl+S` 儲存外，你的工作也會定期自動儲存。

## 主題

從應用程式的主題切換器中，選擇七種內建 UI 主題之一
（包含 GitHub Light/Dark、Dracula、Nord 與 Solarized）。

## 語言（多語系）

介面提供英文與繁體中文兩種語言。可從應用程式設定中切換語言，
立即生效，不需要重新啟動。

## 匯出

Pourdown 的核心價值在於將豐富格式匯入**成** Markdown，而非將 Office 版面完整還原匯出
（畢竟 Markdown 本來就無法表示那樣的版面）。匯出目標是 Markdown 自然對應的兩種
通用、可控版面格式：

- **HTML** —— 用於網頁分享／嵌入
- **PDF** —— 用於標準化的閱讀／列印

刻意不支援 Office 格式匯出（`.docx`／`.xlsx`／`.pptx`）。
